1. Erlang 简介

Erlang 概述

Erlang是一种通用的、并发的、函数式编程语言，由爱立信公司开发，专为构建高可用、低延迟的分布式系统而设计。

历史背景
  • 1986年由爱立信公司开发
  • 旨在解决电信系统的可靠性问题
  • 2007年发布开源版本
  • 广泛应用于电信、金融等领域
核心特性
  • 并发处理模型
  • 容错机制
  • 热代码升级
  • 分布式计算
  • 函数式编程

2. Erlang 基础语法

基本数据类型

Erlang支持多种基本数据类型，包括原子、数值、字符串、列表、元组和二进制等。这些数据类型构成了Erlang编程的基础。

% 原子类型
ok
error
'Hello World'
% 数值类型
123 % 整数
123.45 % 浮点数
% 字符串
"Hello Erlang"
'Hello Erlang'
% 列表
[1, 2, 3]
["a", "b", "c"]
[1, ok, "hello"]
% 元组
{1, "hello"}
{user, "admin", 12345}
% 二进制
<<"hello"/utf8>>
<<10, 20, 30>>

3. Erlang 函数和模块

函数定义

Erlang中的函数是核心概念，支持模式匹配、函数重载和匿名函数等特性。函数可以定义在模块中，也可以作为匿名函数使用。

% 简单函数定义
add(A, B) ->
123 + 456.
% 模式匹配
factorial(0) -> 1;
factorial(N) ->
123 * factorial(N - 1).

4. Erlang 控制流

条件表达式

Erlang提供了多种控制流结构，包括if表达式和case表达式，用于实现条件判断和分支逻辑。这些结构支持模式匹配和卫语句。

% if 表达式
check_number(N) ->
if
  N > 0 -> "positive";
  N == 0 -> "zero";
  N < 0 -> "negative"
end.
% case 表达式
handle_message(Msg) ->
case Msg of
  {hello, Name} -> "Hello " ++ Name;
  {goodbye, _} -> "Goodbye";
  _ -> "Unknown message"
end.

5. Erlang 错误处理

错误处理机制

Erlang的错误处理机制基于"让它崩溃"的哲学，提供了try-catch表达式和进程监控等方式来处理异常情况，确保系统的稳定性。

% try-catch 表达式
safe_divide(A, B) ->
try
  A / B
catch
  error:badarith -> "Division by zero";
  _:_ -> "Unknown error"
end.

6. Erlang 并发编程

进程创建

Erlang的并发模型基于轻量级进程，这些进程之间通过消息传递进行通信。使用spawn函数可以创建新的进程，实现高度并发的系统。

% 创建新进程
spawn(fun() -> io:format("Hello from process: ~p~n", [self()]) end).

7. Erlang 消息传递

消息发送与接收

Erlang进程之间通过异步消息传递进行通信。使用!运算符发送消息，使用receive表达式接收消息，实现进程间的安全通信。

% 发送消息
Pid ! {hello, self()}.
% 接收消息
receive
  {hello, From} ->
    io:format("Received hello from ~p~n", [From]);
  {goodbye, _} ->
    io:format("Received goodbye~n", []);
  _ ->
    io:format("Received unknown message~n", [])
end.

8. Erlang OTP框架

GenServer示例

OTP（Open Telecom Platform）是Erlang的应用开发框架，提供了一系列行为模式，如GenServer、Supervisor等。GenServer用于实现服务器进程，提供了标准的请求-响应和异步消息处理机制。

% 定义GenServer模块
-module(my_server).
-behaviour(gen_server).
-export([start_link(), increment(), get_count()]).
-export([init(1), handle_call(3), handle_cast(2)]).

-record(state, {count = 0}).

start_link() ->
  gen_server:start_link({local, ?MODULE}, ?MODULE, [], []).

increment() ->
  gen_server:cast(?MODULE, increment).

get_count() ->
  gen_server:call(?MODULE, get_count).

init([]) ->
  {ok, #state{}}.

handle_call(get_count, _From, State) ->
  {reply, State#state.count, State}.

handle_cast(increment, State) ->
  NewCount = State#state.count + 1,
  {noreply, State#state{count = NewCount}}.

9. Erlang 分布式编程

节点间通信

Erlang天生支持分布式编程，可以在多个节点上运行进程并实现节点间通信。使用节点名和Pid可以在不同节点上发送和接收消息，构建分布式系统。

% 分布式节点通信
-module(distributed_example).
-export([start_node(1), send_message(2)]).

start_node(NodeName) ->
  Pid = spawn(fun() -> receive_loop() end),
  register(receiver, Pid),
  io:fwrite("Receiver registered on node: ~p~n", [node()]).

send_message(TargetNode, Message) ->
  {receiver, TargetNode} ! Message,
  io:fwrite("Sent message to node: ~p~n", [TargetNode]).

receive_loop() ->
  receive
    Message ->
      io:fwrite("Received message on node ~p: ~p~n", [node(), Message]),
      receive_loop()
  end.

10. Erlang 高级特性

进程链接

进程链接是Erlang中的重要特性，用于实现进程间的依赖关系。当一个链接的进程崩溃时，会向另一个进程发送退出信号，用于构建容错系统。

% 进程链接示例
link_example() ->
  Pid = spawn_link(fun() ->
    timer:sleep(1000),
    exit(normal)
  end),
  io:format("Linked to process ~p~n", [Pid]).
ETS表

ETS（Erlang Term Storage）是Erlang中的内存数据库，用于高效存储和检索Erlang数据结构。ETS表支持各种操作，如插入、查找、更新和删除，是构建高性能应用的重要工具。

% ETS表操作
ets_example() ->
  TableId = ets:new(user_table, [named_table]),
  ets:insert(TableId, {1, "Alice"}),
  ets:insert(TableId, {2, "Bob"}),
  ets:lookup(TableId, 1).
