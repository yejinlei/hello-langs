1. 类型注解 (Type Annotations)

TypeScript - 静态类型，支持类型注解

TypeScript 允许我们为变量、函数参数和返回值添加类型注解，以指定它们的预期类型。

// 变量类型注解
let name: string = "张三";
let age: number = 25;
let isStudent: boolean = true;
// 函数参数和返回值类型注解
function greet(userName: string): string {
    return `Hello, ${userName!`;
}
// 交叉类型
interface Person {
    name: string;
    age: number;
}
interface Employee {
    company: string;
    position: string;
}
type EmployeePerson = Person
// 交叉类型
interface Person {
    name: string;
    age: number;
}
interface Employee {
    id: number;
    department: string;
}
type EmployeePerson = Person & Employee;
const employee: EmployeePerson = {
    name: "John Doe",
    age: 30,
    id: 123,
    department: "IT"
}
// app.ts - 使用命名空间
/// <reference path="geometry.ts" />
let circle = new Geometry.Circle(5);
console.log(`Circle area: ${circle.area()`);
let rectangle = new Geometry.Rectangle(10, 20);
console.log(`Rectangle area: ${rectangle.area()`);
// 嵌套命名空间
namespace MyApp {
    export namespace Utils {
        export function log(message: string): void {
            console.log(`[LOG]: ${message`);
        }
    }
}
MyApp.Utils.log("Hello from nested namespace!");
// 别名导入
import Geo = Geometry;
let geoCircle = new Geo.Circle(10);
console.log(`Circle area (with alias): ${geoCircle.area()`);

命名空间是 TypeScript 早期版本的特性，用于组织代码。在现代 TypeScript 开发中，推荐使用 ES6 模块而不是命名空间，但命名空间仍然可以用于特定场景，特别是当需要将多个文件合并为一个文件时。

12. 装饰器 (Decorators)

TypeScript - 使用装饰器增强类和方法

装饰器是一种特殊类型的声明，它可以被附加到类声明、方法、访问器、属性或参数上。装饰器使用 @expression 语法，其中 expression 必须求值为一个函数，该函数会在运行时被调用，被装饰的声明信息作为参数传入。

要使用装饰器，需要在 tsconfig.json 中启用 experimentalDecorators 选项。

// 类装饰器
function sealed(target: Function) {
    Object.seal(target);
    Object.seal(target.prototype);
}
@sealed
class Person {
    constructor(public name: string, public age: number) {}
}
// 带参数的类装饰器
function classDecorator(params: any) {
    return function(target: Function) {
        console.log(`Class decorator with params: ${JSON.stringify(params)`);
        console.log(`Target class: ${target.name`);
    };
}
@classDecorator({ name: "MyClass", version: 1.0 })
class MyClass {
    message: string;
    constructor() {
        this.message = "Hello";
    }
}
// 索引类型
interface Person {
    name: string;
    age: number;
    location: string;
}
type PersonKeys = keyof Person; // "name" | "age" | "location"
function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
    return obj[key];
};
// 使用命名空间
namespace MyApp {
    export namespace Utils {
        export function log(message: string): void {
            console.log(`[LOG]: ${message`);
        }
    }
}
// 导入命名空间
import Geo = Geometry;
let circle = new Geo.Circle(5);
console.log(`Circle area: ${circle.area()`);

命名空间是 TypeScript 早期版本的特性，用于组织代码。在现代 TypeScript 开发中，推荐使用 ES6 模块而不是命名空间，但命名空间仍然可以用于特定场景，特别是当需要将多个文件合并为一个文件时。

const person: Person = {
    name: "Jane Doe",
    age: 25,
    location: "New York"
};
const name = getProperty(person, "name"); // string
const age = getProperty(person, "age"); // number
// 映射类型
interface User {
    name: string;
    age: number;
}
type ReadonlyUser = { readonly [K in keyof User]: User[K]; };
type PartialUser = { [K in keyof User]?: User[K]; };
const readonlyUser: ReadonlyUser = {
    name: "Bob Smith",
    age: 35
};
const partialUser: PartialUser = {
    name: "Alice Johnson"
};
// 类型参数默认值
interface CacheEntry<T = any> {
    value: T;
    expires?: Date;
};
const stringCache: CacheEntry<string> = {
    value: "Hello World"
};
const anyCache: CacheEntry = {
    value: 42
};

TypeScript 的高级类型系统允许我们构建复杂且灵活的类型定义，以满足各种编程需求。类型别名和可辨识联合类型是其中最常用的高级类型之一。

greet("李四"); // 正确：传入字符串类型
greet(123); // 错误：传入数字类型，预期字符串类型

类型注解使用冒号 : 后跟类型名称。TypeScript 编译器会在编译时检查类型是否匹配。

2. 基本类型 (Basic Types)

TypeScript - 支持多种基本数据类型

TypeScript 包含 JavaScript 的所有基本数据类型，并添加了一些额外的类型用于增强类型系统。

// 字符串类型
let message: string = "Hello TypeScript";
// 数字类型
let count: number = 42;
let price: number = 19.99;
// 布尔类型
let isActive: boolean = true;
// null 和 undefined
let emptyValue: null = null;
let notDefined: undefined = undefined;
// symbol 类型
let uniqueKey: symbol = Symbol("id");
// bigint 类型
let largeNumber: bigint = 9007199254740991n;

TypeScript 还支持 any 类型（允许任何类型）和 void 类型（表示没有返回值）。

3. 联合类型与交叉类型 (Union & Intersection Types)

TypeScript - 支持联合类型和交叉类型

联合类型使用竖线 | 表示一个值可以是多种类型中的一种。交叉类型使用与符号 & 表示一个值同时拥有多种类型的特征。

// 联合类型：number 或 string
let id: number | string;
id = 123; // 正确：number 类型
id = "abc"; // 正确：string 类型
// 联合类型函数
function printId(id: number | string): void {
    if (typeof id === "string") {
        console.log(`String ID: ${id`);
    } else {
        console.log(`Number ID: ${id`);
    }
}
// 交叉类型
interface Person {
    name: string;
    age: number;
}
interface Employee {
    employeeId: string;
    department: string;
}
type EmployeePerson = Person & Employee;
let employee: EmployeePerson = {
    name: "张三",
    age: 30,
    employeeId: "EMP001",
    department: "IT"
};

联合类型和交叉类型使我们能够更灵活地定义类型，满足不同场景的需求。

4. 接口 (Interfaces)

TypeScript - 使用接口定义对象结构

接口用于定义对象的类型，描述对象的形状和结构。接口可以包含属性、方法和索引签名等。

// 基本接口
interface Person {
    name: string;
    age: number;
    isStudent: boolean;
}
let person: Person = {
    name: "李四",
    age: 20,
    isStudent: true
};
// 可选属性
interface Car {
    brand: string;
    model: string;
    color?: string; // 可选属性
}
let car1: Car = {
    brand: "Toyota",
    model: "Camry"
};
// 只读属性
interface Point {
    readonly x: number;
    readonly y: number;
}
let point: Point = { x: 10, y: 20 };
// point.x = 30; // 错误：只读属性不能被修改

接口还可以描述函数类型、索引类型等，是 TypeScript 中定义类型的重要工具。

5. 类 (Classes)

TypeScript - 支持面向对象编程

TypeScript 提供了完整的类支持，包括继承、访问修饰符、抽象类等面向对象特性。

// 基本类
class Person {
    constructor(public name: string, public age: number) {}
    public greet(): string {
        return `Hello, my name is ${this.name`;
    }
}
let person = new Person("张三", 25);
console.log(person.greet()); // 输出: Hello, my name is 张三
// 继承
class Student extends Person {
    constructor(name: string, age: number, public studentId: string) {
        super(name, age);
    }
    public study(): void {
        console.log(`Student ${this.name is studying`);
    }
}
let student = new Student("李四", 20, "S12345");
student.study(); // 输出: Student 李四 is studying
// 访问修饰符
class Employee {
    private salary: number;
    constructor(public name: string, protected department: string) {
        this.salary = 0;
    }
    public setSalary(amount: number): void {
        this.salary = amount;
    }
}

TypeScript 支持三种访问修饰符：public（公开）、protected（保护）和 private（私有）。

6. 泛型 (Generics)

TypeScript - 支持泛型编程

泛型允许我们编写可重用的组件，可以处理多种类型而不是单一类型。泛型提供了类型安全，同时保持代码的灵活性。

// 泛型函数
function identity<T>(value: T): T {
    return value;
}
let numberIdentity = identity<number>(42);
let stringIdentity = identity<string>("Hello");
// 泛型接口
interface Pair<T, U> {
    first: T;
    second: U;
}
let numberStringPair: Pair<number, string> = { first: 1, second: "one" };
// 泛型类
class GenericList<T> {
    private items: T[] = [];
    public add(item: T): void {
        this.items.push(item);
    }
    public get(index: number): T {
        return this.items[index];
    }
}
let numberList = new GenericList<number>();
numberList.add(1);
numberList.add(2);
let firstNumber = numberList.get(0); // 类型为 number

泛型允许我们创建灵活、可重用的组件，同时保持类型安全。泛型可以用于函数、接口、类等多种场景。

7. 类型断言 (Type Assertions)

TypeScript - 使用类型断言明确类型

类型断言允许我们告诉 TypeScript 编译器某个值的具体类型，当我们比编译器更了解值的类型时非常有用。

// 语法 1：尖括号语法
let someValue: any = "this is a string";
let strLength1: number = (<string>someValue).length;
// 语法 2：as 语法（推荐在 TypeScript 中使用）
let strLength2: number = (someValue as string).length;
// 类型断言在 DOM 操作中的应用
let inputElement = document.getElementById("username") as HTMLInputElement;
inputElement.value = "张三"; // TypeScript 现在知道这是一个输入元素
// 双重断言
let x: any = 10;
let y: string = (x as any as string); // 不推荐使用

类型断言只是告诉编译器我们知道的更多信息，它不会在运行时执行任何类型转换。使用类型断言时要小心，确保断言的类型是正确的。

8. 类型守卫 (Type Guards)

TypeScript - 使用类型守卫进行类型检查

类型守卫是一种运行时检查，用于确保值的类型符合预期。TypeScript 支持多种类型守卫，包括 typeof、instanceof、in 和自定义类型守卫函数。

// typeof 类型守卫
function printValue(value: string | number): void {
    if (typeof value === "string") {
        console.log(`字符串: ${value`);
        console.log(`长度: ${value.length`);
    } else {
        console.log(`数字: ${value`);
        console.log(`平方: ${value * value`);
    }
}
// instanceof 类型守卫
class Animal {
    constructor(public name: string) {}
}
class Cat extends Animal {
    meow(): void {
        console.log("Meow!");
    }
}
class Dog extends Animal {
    bark(): void {
        console.log("Woof!");
    }
}
function makeSound(animal: Animal): void {
    if (animal instanceof Cat) {
        animal.meow();
    } else if (animal instanceof Dog) {
        animal.bark();
    } else {
        console.log("Unknown animal sound");
    }
}
// in 类型守卫
interface Bird {
    name: string;
    fly(): void;
}
interface Fish {
    name: string;
    swim(): void;
}
function moveAnimal(animal: Bird | Fish): void {
    if ("fly" in animal) {
        animal.fly();
    } else {
        animal.swim();
    }
}
// 自定义类型守卫
interface Square {
    kind: "square";
    sideLength: number;
}
interface Circle {
    kind: "circle";
    radius: number;
}
type Shape = Square | Circle;
function isSquare(shape: Shape): shape is Square {
    return shape.kind === "square";
}
function getArea(shape: Shape): number {
    if (isSquare(shape)) {
        return shape.sideLength * shape.sideLength;
    } else {
        return Math.PI * shape.radius * shape.radius;
    }
}

类型守卫是 TypeScript 中处理联合类型的重要工具，它可以帮助我们在运行时安全地访问特定类型的属性和方法。

9. 高级类型 (Advanced Types)

TypeScript - 高级类型系统

TypeScript 提供了多种高级类型，用于处理复杂的类型场景。这些高级类型包括类型别名、类型参数默认值、条件类型等。

// 类型别名
type StringOrNumber = string | number;
type User = {
    id: StringOrNumber;
    name: string;
    email: string;
};
// 可辨识联合类型
interface SuccessResponse {
    success: true;
    data: any;
}
interface ErrorResponse {
    success: false;
    error: string;
}
type Response = SuccessResponse | ErrorResponse;
function handleResponse(response: Response): void {
    if (response.success) {
        console.log("Success:", response.data);
    } else {
        console.log("Error:", response.error);
    }
}
// 交叉类型
interface Person {
    name: string;
    age: number;
}
interface Employee {
    employeeId: number;
    department: string;
}
type EmployeePerson = Person & Employee;
let emp: EmployeePerson = {
    name: "John Doe",
    age: 30,
    employeeId: 12345,
    department: "IT"
};
// 索引类型
interface Person {
    name: string;
    age: number;
    address: string;
}
type PersonKeys = keyof Person; // "name" | "age" | "address"
type PersonValues = Person[keyof Person]; // string | number
function getValue<T, K extends keyof T>(obj: T, key: K): T[K] {
    return obj[key];
}
let person: Person = {
    name: "John",
    age: 25,
    address: "123 Main St"
};
let name = getValue(person, "name"); // string
let age = getValue(person, "age"); // number
// 映射类型
interface User {
    name: string;
    age: number;
    email: string;
}
// 只读映射类型
type ReadonlyUser = {
    readonly [P in keyof User]: User[P];
};
// 可选映射类型
type PartialUser = {
    [P in keyof User]?: User[P];
};
// 类型参数默认值
interface CacheEntry<T = any> {
    key: string;
    value: T;
    expiresAt: Date;
}
let stringCache: CacheEntry<string> = {
    key: "name",
    value: "John",
    expiresAt: new Date(2024, 12, 31)
};
let anyCache: CacheEntry = {
    key: "data",
    value: { id: 1, name: "Test" },
    expiresAt: new Date(2024, 12, 31)
}

高级类型是 TypeScript 中强大的特性，它们允许我们创建复杂的类型定义，解决各种类型问题。通过学习和使用这些高级类型，我们可以编写更安全、更灵活的 TypeScript 代码。

10. 模块 (Modules)

TypeScript - 使用模块组织代码

TypeScript 支持使用模块来组织代码，模块可以帮助我们将代码分割成可重用的部分，并控制代码的可见性。TypeScript 使用 ES6 模块语法，包括 import 和 export 语句。

// math.ts - 导出模块
export const PI = 3.14159;
export function add(a: number, b: number): number {
    return a + b;
}
export function multiply(a: number, b: number): number {
    return a * b;
}
export interface Shape {
    area(): number;
}
class Circle implements Shape {
    constructor(private radius: number) {}
    area(): number {
        return PI * this.radius * this.radius;
    }
}
export { Circle };
// app.ts - 导入模块
import { PI, add, multiply, Circle, Shape } from "./math";
console.log(`PI: ${PI`);
console.log(`1 + 2 = ${add(1, 2)`);
console.log(`3 * 4 = ${multiply(3, 4)`);
let circle = new Circle(5);
console.log(`Circle area: ${circle.area()`);
// 重命名导入
import { add as sum } from "./math";
console.log(`Sum: ${sum(5, 6)`);
// 导入所有内容
import * as MathUtils from "./math";
console.log(`PI from MathUtils: ${MathUtils.PI`);

TypeScript 模块支持多种导出方式，包括命名导出和默认导出。模块系统是组织大型 TypeScript 应用程序的重要工具。

11. 命名空间 (Namespaces)

TypeScript - 使用命名空间组织代码

命名空间是 TypeScript 中用于组织代码的另一种方式，它允许我们将相关的代码组合在一起，并避免命名冲突。命名空间可以包含接口、类、函数和变量等。

// geometry.ts
namespace Geometry {
    export const PI = 3.14159;
    export interface Shape {
        area(): number;
    }
    export class Circle implements Shape {
        constructor(private radius: number) {}
        area(): number {
            return PI * this.radius * this.radius;
        }
    }
    export class Rectangle implements Shape {
        constructor(private width: number, private height: number) {}
        area(): number {
            return this.width * this.height;
        }
    }
};
