1. 类型注解 (Type Annotations)

TypeScript 提供了丰富的类型注解功能，用于指定变量、函数参数和返回值的类型。注意事项：类型注解使用冒号(:)语法；TypeScript 会在编译时进行类型检查，帮助发现潜在错误；类型注解不影响运行时性能。

// 基本类型注解
let isDone: boolean = false;
let decimal: number = 6;
let color: string = "blue";
// 数组类型
let list: number[] = [1, 2, 3];
let list2: Array<number> = [1, 2, 3];
// 元组类型
let x: [string, number];
x = ["hello", 10];

2. 接口 (Interfaces)

接口是 TypeScript 中用于定义对象类型的强大工具，它描述了对象的结构和行为。注意事项：接口使用 interface 关键字定义；接口可以包含可选属性、只读属性和方法签名；接口只在编译时起作用，不会出现在运行时代码中。

interface LabeledValue {
    label: string;
}
function printLabel(labeledObj: LabeledValue): void {
    console.log(labeledObj.label);
}
let myObj = { size: 10, label: "Size 10 Object" };
printLabel(myObj);
// 可选属性
interface SquareConfig {
    color?: string;
    width?: number;
}
// 只读属性
interface Point {
    readonly x: number;
    readonly y: number;
}
let p1: Point = { x: 10, y: 20 };
// 函数类型接口
interface SearchFunc {
    (source: string, subString: string): boolean;
}
let mySearch: SearchFunc;
mySearch = function(source, subString) {
    return source.search(subString) > -1;
};

3. 类 (Classes)

TypeScript 支持面向对象编程，提供了完整的类、继承、封装和多态支持。注意事项：类使用 class 关键字定义；可以包含构造函数、属性和方法；支持访问修饰符（public、private、protected）；可以实现接口和继承其他类。

class Greeter {
    greeting: string;
    constructor(message: string) {
        this.greeting = message;
    }
    greet(): string {
        return "Hello, " + this.greeting;
    }
}
let greeter = new Greeter("world");
console.log(greeter.greet());
// 访问修饰符
class Employee {
    public name: string; // 默认
    private salary: number; // 仅类内部可访问
    protected department: string; // 类内部和子类可访问
    constructor(name: string, salary: number, department: string) {
        this.name = name;
        this.salary = salary;
        this.department = department;
    }
    public getInfo(): string {
        return `Name: ${this.name}, Department: ${this.department}`;
    }
}
// 继承
class Manager extends Employee {
    teamSize: number;
    constructor(name: string, salary: number, department: string, teamSize: number) {
        super(name, salary, department); // 调用父类构造函数
        this.teamSize = teamSize;
    }
    public getManagerInfo(): string {
        return this.getInfo() + `, Team Size: ${this.teamSize}`;
    }
}
// 抽象类
abstract class Shape {
    constructor(protected color: string) {}
    abstract getArea(): number; // 抽象方法
    public getColor(): string {
        return this.color;
    }
}
class Circle extends Shape {
    constructor(color: string, private radius: number) {
        super(color);
    }
    getArea(): number {
        return Math.PI * this.radius * this.radius;
    }
}
// instanceof 类型守卫
class Shape {
    abstract getArea(): number;
}
class Circle extends Shape {
    radius: number;
    constructor(radius: number) {
        super();
        this.radius = radius;
    }
    getArea(): number {
        return Math.PI * this.radius ** 2;
    }
}
class Square extends Shape {
    side: number;
    constructor(side: number) {
        super();
        this.side = side;
    }
    getArea(): number {
        return this.side ** 2;
    }
}
function describeShape(shape: Shape): string {
    if (shape instanceof Circle) {
        return `Circle with radius ${shape.radius`;
    } else if (shape instanceof Square) {
        return `Square with side ${shape.side`;
    } else {
        return "Unknown shape";
    }
}
// in 操作符类型守卫
interface Bird {
    fly(): void;
    chirp(): void;
}
interface Fish {
    swim(): void;
    bubble(): void;
}
function moveAnimal(animal: Bird | Fish): void {
    if ("fly" in animal) {
        animal.fly();
    } else {
        animal.swim();
    }
}
// instanceof 类型守卫
class Shape {
    abstract getArea(): number;
}
class Circle extends Shape {
    constructor(public radius: number) { super(); }
    getArea(): number {
        return Math.PI * this.radius ** 2;
    }
}
class Square extends Shape {
    constructor(public side: number) { super(); }
    getArea(): number {
        return this.side ** 2;
    }
}
function printArea(shape: Shape): void {
    console.log(`Area: ${shape.getArea()`);
    if (shape instanceof Circle) {
        console.log(`Radius: ${shape.radius`);
    } else if (shape instanceof Square) {
        console.log(`Side: ${shape.side`);
    }
}
// in 操作符类型守卫
interface Bird {
    fly(): void;
    chirp(): void;
}
interface Fish {
    swim(): void;
    bubble(): void;
}
function moveAnimal(animal: Bird | Fish): void {
    if ("fly" in animal) {
        animal.fly();
    } else {
        animal.swim();
    }
}

4. 泛型 (Generics)

泛型是 TypeScript 中用于创建可复用、类型安全的组件的强大工具。注意事项：泛型使用尖括号(<>)语法定义类型参数；可以在函数、类和接口中使用泛型；泛型提供编译时类型检查，同时保持运行时性能。

function identity<T>(arg: T): T {
    return arg;
}
let output = identity<string>("myString");
let output2 = identity(100); // 类型推断

5. 枚举 (Enums)

枚举是 TypeScript 中用于定义命名常量集合的功能，它提供了一种更友好的方式来表示一组相关的值。注意事项：枚举使用 enum 关键字定义；默认情况下，枚举成员从 0 开始自动编号；可以手动指定枚举成员的值（数字或字符串）；枚举在编译时会被转换为 JavaScript 对象。

enum Color {
    Red,
    Green,
    Blue
}
let c: Color = Color.Green;
// 数字枚举
enum Direction {
    Up = 1,
    Down,
    Left,
    Right
}
// 字符串枚举
enum ColorString {
    Red = "RED",
    Green = "GREEN",
    Blue = "BLUE"
}
let colorName: string = ColorString.Red; // "RED"

6. 高级类型 (Advanced Types)

TypeScript 提供了丰富的高级类型系统，用于解决复杂的类型问题。注意事项：交叉类型（&）将多个类型合并为一个；联合类型（|）表示一个值可以是多种类型之一；类型保护用于在运行时确定联合类型的具体类型；类型别名用于为复杂类型创建更简单的名称。

// 交叉类型
interface A {
    a: number;
}
interface B {
    b: string;
}
type AB = A & B;
let ab: AB = { a: 1, b: "hello" };
// 索引类型
interface Person {
    name: string;
    age: number;
}
let key: keyof Person;
key = "name"; // 有效
key = "age"; // 有效
// 映射类型
type ReadonlyPerson = Readonly<Person>;
type PartialPerson = Partial<Person>;

7. 类型守卫 (Type Guards)

类型守卫是一种在运行时确定联合类型具体类型的技术，它允许 TypeScript 编译器在条件分支中推断更具体的类型。注意事项：使用 typeof 操作符检查基本类型；使用 instanceof 检查类实例；使用 in 操作符检查对象属性；使用类型断言守卫（is 关键字）创建自定义类型检查函数。

// 类型断言守卫
function isString(x: any): x is string {
    return typeof x === "string";
}
function isNumber(x: any): x is number {
    return typeof x === "number";
}
function padLeft(value: string, padding: string | number): string {
    if (isNumber(padding)) {
        return " ".repeat(padding) + value;
    } else if (isString(padding)) {
        return padding + value;
    } else {
        throw new Error("Invalid padding type");
    }
}

8. 模块 (Modules)

模块是 TypeScript 中用于组织代码的机制，它允许将代码分割成多个文件，提高代码的可维护性和复用性。注意事项：使用 export 关键字导出接口、类、函数等；使用 import 关键字导入其他模块的导出内容；模块可以重命名导入和导出的内容；默认导出和命名导出可以同时使用。

// module1.ts
export interface User {
    id: number;
    name: string;
}
export function greet(user: User): string {
    return `Hello, ${user.name!`;
}
// module2.ts
import { User, greet } from "./module1";
let user: User = { id: 1, name: "Alice" };
console.log(greet(user));

9. 命名空间 (Namespaces)

命名空间是 TypeScript 中用于组织代码的另一种方式，它允许将相关的代码组织在一起，避免命名冲突。注意事项：使用 namespace 关键字定义命名空间；需要使用 export 关键字导出命名空间中的内容才能在外部访问；命名空间可以嵌套；命名空间主要用于内部代码组织，现代 TypeScript 项目通常更推荐使用 ES 模块。

namespace MyApp {
    export interface User {
        id: number;
        name: string;
    }
    export namespace Utils {
        export function log(message: string): void {
            console.log(`[LOG]: ${message`);
        }
    }
}
import MyApp = MyApp;
let user: MyApp.User = { id: 1, name: "Alice" };
MyApp.Utils.log(`User: ${user.name`);

10. 装饰器 (Decorators)

装饰器是 TypeScript 中用于修改类、方法、属性或参数行为的高级特性。注意事项：装饰器使用 @ 语法应用；需要在 tsconfig.json 中启用 experimentalDecorators 选项；可以用于实现元编程、依赖注入、日志记录等功能；装饰器在编译时执行，而不是运行时。

// 类装饰器
function sealed(constructor: Function) {
    Object.seal(constructor);
    Object.seal(constructor.prototype);
}
@sealed
class Person {
    constructor(public name: string) {}
}
// 参数化装饰器
function logged(target: any, propertyName: string) {
    console.log(`Property ${propertyName was accessed.`);
}
class Employee {
    @logged
    constructor(private _name: string) {}
    get name(): string {
        return this._name;
    }
}

11. 声明文件 (Declaration Files)

声明文件是 TypeScript 中用于描述 JavaScript 库或模块类型的特殊文件，扩展名为 .d.ts。注意事项：声明文件不包含实现代码，只包含类型定义；使用 declare 关键字声明类型；可以为第三方 JavaScript 库编写声明文件；npm 上的 @types/* 包提供了许多流行库的声明文件。

// my-module.d.ts
declare module "my-module" {
    export function greet(name: string): string;
    export interface Options {
        debug?: boolean;
    }
}

12. 编译配置 (Compilation)

tsconfig.json 是 TypeScript 项目的编译配置文件，用于控制 TypeScript 编译器的行为。注意事项：tsconfig.json 位于项目根目录；compilerOptions 部分包含主要配置选项；target 选项指定编译后的 JavaScript 版本；module 选项指定模块系统；outDir 选项指定输出目录；strict 选项启用严格类型检查。

// tsconfig.json
{
    "compilerOptions": {
        "target": "ES2020",
        "module": "commonjs",
        "strict": true,
        "esModuleInterop": true,
        "skipLibCheck": true,
        "forceConsistentCasingInFileNames": true
    }
}

13. 高级概念 (Advanced Concepts)

TypeScript 提供了一系列高级类型特性，用于创建更复杂和灵活的类型定义。注意事项：条件类型允许根据条件选择类型；映射类型用于转换现有类型；模板字面量类型用于创建字符串字面量类型；高级类型可以组合使用，创建强大的类型系统。

// 条件类型
type IsString<T> = T extends string ? true : false;
let strType: IsString<string> = true;
let numType: IsString<number> = false;
// 条件类型推断
type ReturnType<T extends (...args: any[]) => any> = T extends (...args: any[]) => infer R ? R : never;
function f(x: number): string {
    return x.toString();
}
let r: ReturnType<typeof f> = "hello";

15. Node.js 与 TypeScript

TypeScript 可以与 Node.js 完美结合，提供类型安全的服务器端开发体验。注意事项：需要安装 @types/node 获取 Node.js 类型定义；使用 tsconfig.json 配置编译选项；Express 等框架也有对应的类型定义包（如 @types/express）。

import express from 'express';
import { Request, Response } from 'express';
const app = express();
const PORT = 3000;
interface User {
    id: number;
    name: string;
}
const users: User[] = [
    { id: 1, name: 'Alice' },
    { id: 2, name: 'Bob' }
];
app.get('/users', (req: Request, res: Response) => {
    res.json(users);
});
app.listen(PORT, () => {
    console.log(`Server is running on port ${PORT`);
});

16. 测试 (Testing)

TypeScript 支持多种测试框架，如 Jest、Mocha 等，提供类型安全的测试编写体验。注意事项：使用 @types/jest 等类型定义包；类型系统可以帮助捕获测试中的潜在错误；可以为测试编写自定义类型守卫和断言函数。

// calculator.ts
export function add(a: number, b: number): number {
    return a + b;
}
export function subtract(a: number, b: number): number {
    return a - b;
}
// calculator.test.ts
import { add, subtract } from './calculator';
import { describe, it, expect } from 'jest';
describe('Calculator', () => {
    it('should add two numbers', () => {
        expect(add(1, 2)).toBe(3);
    });
    it('should subtract two numbers', () => {
        expect(subtract(5, 3)).toBe(2);
    });
});

17. 性能优化 (Performance)

TypeScript 提供了多种性能优化技术，帮助构建高效的应用程序。注意事项：优先使用接口进行对象类型定义（接口支持合并，类型别名不支持）；避免不必要的类型断言；合理使用泛型和高级类型，避免过度复杂的类型计算影响编译性能。

// 接口合并 vs 类型别名
// 优先使用接口进行对象类型定义
interface User {
    id: number;
}
interface User {
    name: string;
}
// 避免不必要的类型断言
// 好的做法
let x: unknown = "hello";
if (typeof x === "string") {
    console.log(x.length); // 不需要类型断言
}

18. 最佳实践 (Best Practices)

TypeScript 开发的最佳实践有助于提高代码质量和可维护性。注意事项：使用明确的类型注解，避免过度依赖类型推断；尽量避免使用 any 类型；使用接口定义对象结构，提高代码可读性；合理使用泛型和高级类型，增强代码复用性。

// 使用明确的类型注解
// 好的做法
let name: string = "Alice";
// 避免使用 any
// 不好的做法
// let name: any = "Alice";
// 使用接口定义对象结构
interface Product {
    id: number;
    name: string;
    price: number;
}
let product: Product = { id: 1, name: "Laptop", price: 999 };

19. 常见错误 (Common Errors)

TypeScript 开发中常见的错误类型和解决方案。注意事项：类型不匹配是最常见的错误，需要确保变量类型与使用方式一致；避免访问对象不存在的属性，可以使用可选链操作符 (?.) 或类型断言；注意区分接口和类型别名的使用场景。

// 错误：类型不匹配
// let age: number = "25"; // 类型错误
// 错误：访问不存在的属性
interface User {
    id: number;
}
let user: User = { id: 1 };
// console.log(user.name); // 错误：name 属性不存在
// 正确的做法
interface User {
    id: number;
    name?: string; // 可选属性
}
